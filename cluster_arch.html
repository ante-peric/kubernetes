4. Cluster Architecture


Kubernetes Architecture se sastoji od dvije ili više radne stanice.

Svaki Kubernetes Cluster se sastoji od:

  • CONTROL NODE
Upravlja Worker radnim stanicama, planira koji container ide gdje, planira raspored zadataka I slično.
On sadrži više komponenti pomoću kojih upravlja cijelim Clusterom.
  • Kube-apiserver– Glavna komponenta koja upravlja svim komponentama Cluster – a
  • ETCD Cluster– Služi za pohranu podataka u key-value formatu
  • kube-scheduler- Služi za upravljanje I identifikaciju containera
  • Controller Manager:

    1. Node – Controller
    2. Replication – Controller
  • WORKER NODE
  • kubelet– Sluša naredbe kube-apiserver te odrađuje zadatke kao kreiranje I uništavanje container – a
  • Kube – proxy • – Služi za komunikaciju između worker radnih stanica
