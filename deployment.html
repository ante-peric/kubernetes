4.10. Deployment


U slučaju da nam je potrebna nadogradnja aplikacije ili u slučaju da nam nadogradnja nije uspjela moramo napraviti “roll-back”, undo changes unutar containera I slično.
Za ovakav scenarij možemo koristiti “deployment”. Deployment je Kubernetes objekt koji je na većoj razini u Kubernetes arhitekturi.

Deployment se kreira na način vrlo slično kao ReplicaSet osim što u dijelu:

kind: ReplicaSet

Koristimo

kind:Deployment

Kreira se na isti način:

kubectl create -f sample.yaml

Na ovaj način se I automatski kreira ReplicaSet kao I pod – I.

Naredba za ispis svih kreiranih objekata:

kubectl get all

Dodatak.

Kreiramo template ako imamo problema s memoriranjem yaml syntaxe:

POD

kubectl run nginx --image=nginx --dry-run=client -o yaml

Deployment

kubectl create deployment nginx --image=nginx --dry-run=client -o yaml
