4.1. Docker vs containerD


U Kubernetes Cluster – možemo koristit više “Runtime” oruđa kao što su:
  • rkt
  • docker
U razvitku Kubernetes tehnologije došlo je do nekih promjena kod samih proizvođača I runtime tehnologije koje poštuju pravilo CRI – Container Runtime Interface mogu biti implementirane u Kubernetes.
Pravilo se zove OCI – Open Container Initiative I sadrži dvije stavke:
  • imagespec
  • runtimespec
Doker nije gradjen da podrži ove standarde pa je Kubernetes zajednica kreirala privremeno rješenje preko “dockershim” tehnologije da se nastavi korištenje Docker techologije.
Ali ovo rješenje je bilo privremeno. Nakon što je Docker kreirao tehnologiju koja za je unaprijedila kreiranje, upravljanje I slično I nazvana containerd moglo se nastaviti sa korištanjem Kubernetesa.
Iako je containerd dio Docker tehnoligije može se koristiti odvojeno od Docker tehnologije.
Dakle, može se instalirati containerd runtime I upravljati containerima.

Containerd dolazi s sljedećim command line alatima:
  • ctr
  • nerdctl
  • crictl
ctr – nije user frendly I koristi se rijetko

Alternativa je korištenje nerdctl command line koji ima skoro pa sve naredbe slične docker naredbi

Jos jedan command utillty koji je kreiran od strane Kubernetes je crictl I pomoću ovog oruđa možemo koristiti naredbe između različitih runtime proizvođača.
Instalira se posebno I koristi se većinim za debugging.

Lokacija end – point

unix:///run/crio/crio.sock
