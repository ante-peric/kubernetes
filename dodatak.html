4.13. Dodatak


replace, edit, scale.

replace schema:

1.

vim mainfest.yml

kubectl replace -f mainfest.yml

edit schema:

1.

kubectl edit -f mainfest.yml

2.

kubectl edit replicasets.apps replicaset-1

scale schema:

1.

kubectl scale --replicas=6 -f mainfest.yml

2.

kubectl scale replicasets.apps replicaset-1 --replicas=7

Imperative – korak po korak instrukcije - mainfest.yml file

Declerative – samo specificiranje finalnog cilja -run, expose, scale, delete, ...

Imperative Naredbe:

POD

POD Kreiranje:

kubectl run nginx --image=nginx

POD Mainfest YAML

kubectl run nginx --image=nginx --dry-run=client -o yaml

Deployment

Deployment Kreiranje:

kubectl create deployment --image=nginx nginx

Deployment Mainfest YAML:

kubectl create deployment --image=nginx nginx --dry-run=client -o yaml

Deployment Mainfest YAML s replicas:

kubectl create deployment --image=nginx nginx --dry-run=client -o yaml replicas=4

Deployment scale:

kubect scale deployment nginx --replicas=4

Services

ClusterIP

kubectl expose pod redis --port=6379 --name redis-service --dry-run=client -o yaml

ili

kubectl create service clusterip redis --tcp=6379:6379 --dry-run=client -o yaml

NodePort

kubectl expose pod nginx --type=NodePort --port=80 --name nginx-service --dry-run=client -o ymal

ili

kubectl create service nodeport nginx --tcp=80:80 --node-port=30008 --dry-run=client -o ymal
