4.12. Namesapces


U dosadšnjem dijelu kreirali smo različite objekte ako što su, pod, ReplicationControler, ReplicaSet, Deployment, Services. Svi ovi objekti su kreirani u jednom Namespace.

Kubernetes starta s 4 inicijalna namespace’s:

Namespace Opis
default Kubernetes uključuje ovaj namespace tako da se može odmah koristiti kreirani Cluster bez da se kreira novi namespace
kube-node-lease Ovaj namespace drži “Lease” objekte vezane za svaki node.
kube-public Ovaj namespace je rezerviran za Cluster korištenje u slučaju da se neki od resursa moraju postaviti kao vidljivi
kube-system Namespace za objekte kreirane od strane Kubernetes sistema.

Tablica 26. Lista Defaultnih Namespace

Ispis svih namespace

kubectl get namespace

Kreiramo novi namespace imena “dev”:

vim namespace-dev.yml

apiVersion: v1
kind: Namespace
metadata:
  name: dev


kubectl create -f namespace-dev.yml

Drugu način je preko command line:

kubectl create namespace prod

Dodavanje pod – a u određeni namespace

Imao već gotov file

apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
    tier: frontend

spec:
  containers:
  - name: nginx
    image: nginx


kubectl create -f sample.yaml --namespace=dev


apiVersion: v1
kind: Pod
metadata:
  name: nginx
  namespace: dev
  labels:
    app: nginx
    tier: frontend

spec:
  containers:
    - name: nginx
      image: nginx


Uz kreiranje namespace kreira se I DNS automatski. Pod ima dns record kao:

ime_pod.namespace.service.cluster.local

Naredba
kubectl get pods


Ako želimo ispisati podove u određenom namespace:

kubectl get pods --namespace=dev


Uvijek ispisuje informacije iz default namespace. Ako želimo postaviti da nam ispisuje iz nekog drugog namespace po postavci.

kubectl config set-context $(kubectl config current-context) --namespace=dev


Ispis svih podova u svim namespace

kubectl get pods --all-namespaces

kubectl get pods -A
