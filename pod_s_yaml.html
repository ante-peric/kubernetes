4.8.1. POD sa yaml


Pretpostavimo da je Cluster kreiran I pokrenut. Za kreiranje container – a tj POD to možemo odraditi I s file – m.
Sintaksa je pisana u yaml formatu I ovo je također jednostavan primjer gdje ćemo kreirati NGINX servis.
Tanent je podešen kao u dijelu 3.8.

Kubernetes prati “hard – coded” strukturu file – a. Imamo 4 osnovna top level polja koja moraju postojati:
  • apiVersion
  • kind
  • metadata
  • spec:

Syntax file – a.

apiVersion:
kind:
metadata:

spec:

Tablica 13. POD syntax file


apiVersion – String Vrijednost Označava verziju Kubernetes API za kreiranje objekta

Tablica 14. POD apiVersion

Mogućnosti:

apiVersion
POD v1
Service v1
ReplicaSet apps/v1
Deployment apps/v1

Tablica 15. Pod apiVersion mogućnosti


kind – String Vrijednost Označava tip objekta kojeg pokušavamo kreirati kao što je POD, Service, ReplicaSet, Deployment

Tablica 16. pod – kind


metadata – Form of Dictionary Polje koje je hierarhijski poviš objekta I može sadržavati “child” polja

Tablica 17. pod metadata

Primjer:

metadata:
 name: myapp-pod
 labels:
  app:myapp
  type: fron-end


name: -property (dictionary)
labels: - property (dictionaty)		 Property name I labels su child od metadata I moraju biti u istoj liniji jer su “bliznaci”

Tablica 18. pod metadata opis


spec: Polje gdje se kreiraju objekti


spec:
  containers:
  - name: nginx-container
   image: ngnix

Tablica 19. pod spec


conatiner je List/Array
"-" znači da je prvi u List/Array

Primjer:

$ mkdir project
$ cd project/


Ako ne možemo isprve se sjetint syntax imamo naredbu koja nam kreira privremeni file, naredba koja provjerava sintax I ne čini nikakve radnje:

$ kubectl run web --image=nginx --dry-run -o yaml

W1019 15:57:01.737266 42333 helpers.go:692] --dry-run is deprecated and can be replaced with --dry-run=client.
apiVersion: v1
kind: Pod
metadata:
creationTimestamp: null
labels:
run: web
name: web
spec:
containers:
- image: nginx
name: web

dnsPolicy: ClusterFirst
restartPolicy: Always
status: {}



Na ovaj način vidimo kako bi nam izgledao naš .yaml file I ovaj output možemo usmjeriti u file s naredbom.

$ kubectl run webapp --image=nginx --dry-run=client -o yaml > sample.yaml


Mi možemo kreirati I ručno .yaml file na način:

$ vim pod.yml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
    tier: frontend

spec:
  containers:
  - name: nginx
    image: nginx



Ako želimo dodati jos jedan container

  containers:
  - name: nginx
    image: nginx
  - name: busybox
    image: busybox



Sad možemo kreirati POD

[ante@controlkube project]$ kubectl create -f pod.yml

pod/nginx created


Može se koristiti I apply naredba

$ kubectl apply -f pod.yml



Provjera detalja kreiranog POD

$ kubectl describe pod nginx
