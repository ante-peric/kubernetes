4.9.1. Replication Controller


Replication Controller također mora sadržavati 4 osnovne sekcije kao I POD.
apiVersion:
kind:
metadata:
spec:
apiVersion je podržan kao “v1”.
kind je podržan kao Replication Controller

Kreiramo primjer:
apiVersion: v1
kind: ReplicationController
metadata:
  name: telefonski-imenik-repliction-controller
  labels:
    app: tel-imenik-app
    type: jsp

Ovaj dio je sličan kao I kod PODS. Najkritičniji dio je spec: Prvo se kreira “template” sekciju unutar spec sekcije.

Primjer:

spec:
  template:
    POD


Unutar POD dijela dodajemo kao što smo dodavali kod kreiranja POD. Dodajemo cijeli dio metadata I spec kao child od “template”

metadata:
  name: myapplication-rc
  labels:
    app: moja_aplikacija
    type: front-end

spec:
  containers:
  - name: nginx-container
    image: nginx

Tako da dobijemo sljedeće unutar “template” kao “child”

spec:
  template:
    metadata:
      name: telefonski-imeni
      labels:
        app: telefonski-imenik-app
        type: jsp

    spec:
      containers:
      - name: nginx-container
        image: nginx

Sad imamo file kao ovaj:
apiVersion: v1
kind: ReplicationController
metadata:
  name: telefonski-imenik-repliction-controller
  labels:
    app: tel-imenik-app
    type: jsp

spec:
  template:
    metadata:
      name: telefonski-imeni
      labels:
        app: telefonski-imenik-app
        type: jsp

    spec:
      containers:
      - name: nginx-container
        image: nginx

Za kreiranje Replication Controler dodajemo još jedno svojstvo “replicas”

spec:
  template
  |
  |
  |
  replicas: 3


Broj tri znači da ćemo imati tri (identočna pod – a). Imamo finalni file.

apiVersion: v1
kind: ReplicationController
metadata:
  name: telefonski-imenik-repliction-controller
  labels:
    app: tel-imenik-app
    type: jsp

spec:
  template:
    metadata:
      name: telefonski-imenik
      labels:
        app: telefonski-imenik-app
        type: jsp

    spec:
      containers:
      - name: nginx-container
        image: nginx

    replicas: 3


Kreiramo Replication Controller. (ovo je u fileu naziva:rc.yml)

kubectl create -f rs.yml
