4.9.2.1. Scale ReplicaSet


Zamislimo scenarij da nam je potreban veći broj POD jer se povećava broj korisnika. Za taj scenarij možemo Scale ReplicaSet.

Jedan od načina je da povećamo broj u replica setu za broj koliko nam je potrebno. Uređujemo file

replicas: 3

u

replicas: 6

Ako želimo 6 POD – a. Zatim pokrećemo naredbu
kubectl replace -f sample.yml

Drugin način je pokretanje naredbe:

kubectl scale --replicas=6 -f sample.yml

Treći način

kubectl scale --replicas=6 replicaset password-keeper-replicaset

replicaset type
password-keeper-replicaset Ime


NAPOMENA:

Drugi I treći način neće napraviti update file – a.

Četvrti način:

Koristimo edit naredbu:

kubectl edit rs password-keeper-replicaset

Ova naredba otvara file gdje uređujemo
