3.2. Persistent Volume - PV


Svaki korisnik koji će koristiti Persistent Volume zbog jednostavnosi uveden je pojam Persistent Volume Claim.

Persistent Volume se kreira na način:

vim pv-definition.yml

apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv-vol1
spec:
  accessModes:
    - ReadWriteOnce

  capacity:
    storage: 1Gi

  hostPath:
    path: /tmp/data

Access Modes:

• ReadOnlyMany - Ovaj mode omogućava da se volume učita kao read only za više nodova.

• ReadWriteOnce - Ovaj mode omogućava da se volume učita kao read write za jedan node.

• ReadWriteMay - Ovaj mode omogućava da se volume učita kao read write za jedan node.

kubectl create -f pv.yml

Provjera:

kubectl get persistentvolume

Brisanje:

kubectl delete persistentvolume pv-vol1
